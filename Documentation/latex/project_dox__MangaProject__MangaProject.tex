\item \mbox{[}tag\mbox{]} Case selector tag
\item \mbox{[}spec\mbox{]} Simple tag with one value only
\end{DoxyItemize}

Max and min values define the interpolation range for the salinity.

salinity\+\_\+tree = ((x\+\_\+tree -\/ min\+\_\+x) / (max\+\_\+x -\/ min\+\_\+x) $\ast$ (salinity\+\_\+max -\/ salinity\+\_\+min) + salinity\+\_\+min) \hypertarget{project_dox__MangaProject__tree_dynamics__belowground_competition__FixedSalinity__salinty}{}\doxysubsection{\quad{}\mbox{[}spec\mbox{]}salinty}\label{project_dox__MangaProject__tree_dynamics__belowground_competition__FixedSalinity__salinty}
There are two ways to indicate the salinity of the bottom water\+: ~\newline
 (i) two values, each for the location defined via max\+\_\+x and min\+\_\+x (option 1), or ~\newline
 (ii) a time series with values for these (option 2).

Option (i)$\ast$$\ast$

Salinity in kg/kg. e.\+g. 35 ppt corresponds to 0.\+035. Two values (min and max) need to be specified.

Example\+: ~\newline
 $<$salinity$>$ 0.\+005 0.\+095 $<$/salinity$>$$\ast$

~\newline
 Option (ii)$\ast$$\ast$

Path to file containing salinity in kg/kg. Two values (min and max) need to be specified for each time step you want to define the salinity. For time steps with no defined salinity, values will be linearly interpolated. The input file needs to be a csv file with semicolon as separator and 3 columns, defining the time step (in seconds) and salinity on the left and right boundary (in kg/kg).

Note\+: the first row will be skipped (as we expect the header to be present). ~\newline
 Example xml-\/file\+:

$<$salinity$>$ test/\+Small\+Tests/input\+Files/salinity\+\_\+\+A.\+csv $<$/salinity$>$$\ast$

Example csv file\+:

t\+\_\+step;salinity\+\_\+1;salinity\+\_\+2~\newline
 0;0.\+010;0.\+020~\newline
 1000000;0.\+011;0.\+021~\newline
 2000000;0.\+012;0.\+022~\newline

\item Group formation
\item Initialization of the root graft formation process
\item Calculation of below-\/ground resources
\end{DoxyItemize}

\doxysubsection*{Group formation}

In this sub-\/model the formation of is groups of connected trees in managed. Based on tree variables ({\itshape partners}), a graph dictionary and a link list a generated.

\doxysubsection*{Initialization rgf process}

The formation of root grafts between a pair of trees will be initialized when the following conditions are met


\begin{DoxyItemize}
\item the root plates of trees overlap (i.\+e. distance between trees $<$ sum of root radii)
\item the contact probability (i.\+e. 1 -\/ distance/r\+\_\+root\+\_\+ij) is smaller than a randomly generated number
\item the trees are not already connected with one another
\item the trees are currently not in the process of rgf with one another
\end{DoxyItemize}

\doxysubsection*{Below-\/ground resources}

In this concept, three volume flows describe the water flow in the tree\+:


\begin{DoxyItemize}
\item {\itshape water\+\_\+absorbed} that is the water absorbed by the tree from the soil column
\item {\itshape water\+\_\+available} that is the water available for tree growth and maintenance, which corresponds to water uptake in the Simple\+Bettina model
\item {\itshape water\+\_\+exchanged} that is the water exchanged between connected trees
\end{DoxyItemize}

The volume flows are calculated utilizing the electronic-\/hydraulic analogy. This allows the application of the Kirchhoff\textquotesingle{}s laws, resulting in a set of linear equations.

~\newline


This concept returns a multiplier for water loss or gain. This class is the basis (parent) for further Network concepts such as Network\+Fixed\+Salinity.

Mortality concepts define why trees die. ~\newline
 The following concepts are available for {\ttfamily Simple\+Bettina} and {\ttfamily Network\+Bettina}


\begin{DoxyItemize}
\item {\ttfamily No\+Growth}\+: trees die if growth is $<$= 0
\item {\ttfamily Random}\+: trees die randomly with a defined probability
\item {\ttfamily Random\+Growth}\+: trees die randomly with a biomass-\/increment-\/dependent probability, thus, the probability to die increases with tree age ~\newline

\item {\ttfamily Memory}\+: trees die if relative growth falls below a given threshold
\end{DoxyItemize}

Mortality concepts are defined using the tag {\ttfamily $<$mortality$>$}. The default concept is {\ttfamily No\+Growth} (no specification in the input xml file required). Concepts can be combined by listing them in {\ttfamily $<$mortality$>$}, separated by a whitespace, e.\+g. {\ttfamily $<$mortality$>$ Random Memory $<$/mortality$>$}.

{\ttfamily Memory} is a mortality concept.

In {\ttfamily Memory}, a tree remembers its relative growth of the preceding N years (in m³ per memory period). Relative growth is defined as the fraction of growth in tree biomass (in m³ per m³). If the relative growth falls below a certain threshold, a tree dies. ~\newline


The default threshold is 0.\+5 \% (i.\+e. {\ttfamily $<$threshold$>$ 0.\+005 $<$/threshold$>$}). ~\newline
 The default memory period is 1 year (i.\+e. {\ttfamily $<$period$>$ 31557600 $<$/period$>$}).

This concept is based on mechanistic causes, e.\+g. slowing down of growth due external perturbations. A tree can survive a period of resource scarcity but if conditions do not improve, the tree dies. ~\newline
